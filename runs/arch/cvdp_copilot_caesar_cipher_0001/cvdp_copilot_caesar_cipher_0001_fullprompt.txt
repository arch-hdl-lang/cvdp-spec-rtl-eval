Natural-language specification:
Design the System Verilog module `caesar_cipher` to implement a Caesar cipher for character encryption. The module should take an 8-bit ASCII character as input and produce a new character based on a 4-bit key value. The module should handle both uppercase (A-Z) and lowercase (a-z) letters while leaving non-alphabetic characters unchanged.

### Interface

#### Inputs:

- **input_char ([7:0], 8-bit)**: The ASCII character to be shifted. It should be within the range of standard ASCII values.
- **key ([3:0], 4-bit)**: The shift key. A 4-bit value (0-15) determines the shift applied to alphabetic characters.

#### Output:

- **output_char ([7:0], 8-bit)**: The resulting ASCII character after applying the Caesar cipher shift, respecting character casing. Non-alphabetic characters remain unchanged.

## Design Specification:

- **Uppercase Letters**: If the input is an uppercase letter ('A'-'Z'), the character should be modified within the uppercase range.
- **Lowercase Letters**: If the input is a lowercase letter ('a'-'z'), the character should be modified within the lowercase range.
- **Non-alphabetic Characters**: Characters that are not within 'A'-'Z' or 'a'-'z' should remain unchanged in the output.
- **Modulo Operation**: The shifting must wrap around within the alphabet using modulo 26, ensuring characters like 'Z' roll back to 'A' and 'z' to 'a' when shifted.
- Design should follow combinational logic

### Simplified Algorithm:

1. **Check** if `input_char` is an uppercase letter (A-Z):
   - Shift the character by `key` positions within the range A-Z.
2. **Check** if `input_char` is a lowercase letter (a-z):
   - Shift the character by `key` positions within the range a-z.
3. **If not alphabetic**, leave `input_char` unchanged.

Assume `input_char` = 'h' (`01101000`) and `key` = 3 (`0011`).

1. **Lowercase Handling**:
