Problem ID: cvdp_copilot_16qam_mapper_0001

Natural-language specification:
Design a combinational RTL (using SystemVerilog) module to implement a QAM16 mapper with interpolation. The module maps input bits into QAM16 symbols, computes interpolated values between adjacent symbols, and outputs real (`I`) and imaginary (`Q`) components of the mapped and interpolated values.

### Module Name:
`qam16_mapper_interpolated`

The module is parameterized for flexibility in the number of input symbols while maintaining fixed bit widths for inputs and outputs to support QAM16 mapping.

---

## Parameters

# Parameter Table

| Parameter   | Description                            | Default | Constraints             |
|-------------|----------------------------------------|---------|-------------------------|
| `N`         | Number of input symbols               | 4       | ≥ 2 and multiple of 2 (e.g., 2, 4, 6, 8, etc.) |
| `IN_WIDTH`  | Bit width of each input symbol (fixed) | 4       | Fixed at 4              |
| `OUT_WIDTH` | Bit width of the output components (fixed) | 3   | Fixed at 3              |

### Inputs

|-------------|---------------------------|-------------------------------------------------------------------------------------------------|
| `bits`      | `N*IN_WIDTH`              | Packed input bits. Each group of 4 bits (of width `IN_WIDTH`) represents a QAM16 symbol. Any combination of 4 bits is valid. The input is a packed set of `N` entries, each of `IN_WIDTH` (4 bits). |

### Outputs

| Name        | Width                     | Description                                                                                                               |
|-------------|---------------------------|---------------------------------------------------------------------------------------------------------------------------|

| `Q`         | `(N + N/2)*OUT_WIDTH`     | Packed output of imaginary (Q) components, including mapped and interpolated values. Each `OUT_WIDTH` interval represents a signed integer. The size is automatically determined based on the existing parameters `N` and `OUT_WIDTH`. |

## Behavioral Definitions

1. **Mapping Input Bits**:
   - Each input symbol is divided into two distinct parts, based on the bit positions within the `IN_WIDTH` group:
     - **MSBs** (most significant 2 bits): These correspond to bits `[IN_WIDTH-1 : IN_WIDTH-2]` of the symbol and are mapped to the `I` (real) component using the following values:
       - `00`: `-3`
       - `01`: `-1`
       - `10`: `1`
       - `11`: `3`
     - **LSBs** (least significant 2 bits): These correspond to bits `[1 : 0]` of the symbol and are mapped to the `Q` (imaginary) component using the same values as above.
   - This process is repeated for all `N` input symbols, with each symbol being extracted sequentially from the packed `bits` input vector.

2. **Interpolation**:
   - For every two consecutive symbols, compute the interpolated values for the `I` and `Q` components using a simple arithmetic mean:
     - **Interpolated `I`**: This is the result of adding the `I` components of the two symbols and dividing the sum by 2.
     - **Interpolated `Q`**: This is the result of adding the `Q` components of the two symbols and dividing the sum by 2.
   - This process is repeated for all pairs of consecutive input symbols.
   - The signals holding the interpolated samples need to have 1 bit more than `OUT_WIDTH` to accommodate the addition of two signed `OUT_WIDTH` signals.

3. **Output Arrangement**:
   - The outputs `I` and `Q` are arranged in the following order:
     - First mapped symbol
     - Interpolated value
     - Second mapped symbol
   - This pattern repeats for every pair of input symbols.

## Timing and Synchronization
- The module operates entirely combinationally, with no clock or reset inputs.
- Outputs are updated based on the current values of the input bits.
