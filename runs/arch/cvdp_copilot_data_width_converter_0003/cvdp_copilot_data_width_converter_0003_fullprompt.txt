## Design Specification:

**Module Name:**  
`data_width_converter`

### Key Considerations:
- The design should handle sequential 32-bit data inputs and combine them into a single 128-bit output.
- The module operates on the positive edge of the clock signal.
- The output should be generated once four valid 32-bit inputs have been received.
- The output must be flagged as valid with the `o_data_out_valid` signal.
- The system should reset when the `reset` signal is asserted, clearing any internal states.

### Interface:

**Inputs:**
- `clk`:  Data operations will be performed during positive edge of this clock. The clock operates at 50:50 dutycycle.
- `reset`: Asynchronous Active-high reset signal to initialize the system.
- `data_in` (32bits,[31:0]): 32-bit data input to be concatenated into the output.
- `data_valid`: A signal indicating whether the current `data_in` is valid.

**Outputs:**
- `o_data_out` (128-bits,[127:0]): 128-bit concatenated data output, generated after four valid 32-bit inputs.
- `o_data_out_valid`: A signal indicating that the output data is valid.

### Edge Cases:
- **Reset Condition:** When `reset` is asserted, all internal states and buffers should be cleared, and no output should be produced until valid inputs are received.
- **Incomplete Data:** If fewer than four valid inputs are received before `reset` is asserted, no output is generated, and the system waits for further valid inputs.
- **Continuous Operation:** After producing a valid 128-bit output, the system will reset its internal counters and buffers to prepare for the next set of four 32-bit inputs.
